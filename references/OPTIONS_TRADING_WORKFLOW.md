# 期权策略交易流程

当用户涉及期权交易、期权机会相关问题时，使用此流程。

## 第一步：查询策略详细信息和交易订单，以及策略规则

所有期权交易、期权机会相关问题，**必须先查询到策略详细信息和交易订单，以及策略规则**：

### 查找交易策略详细信息和交易订单

- 如果用户没有提供具体的策略，默认通过`queryAllStrategy`查询所有策略，然后进行匹配，无法匹配的情况需要用户来确认具体的策略。
- 基于用户提供的策略或匹配到的策略，调用`queryStrategyDetailAndOrders`查询策略详细信息和交易订单。

### 学习策略规则

调用 `queryStrategyRule` 工具查询相关交易策略规则：
- 通过`queryStrategyDetailAndOrders`返回确认策略类型，指定 `strategyCode` 可查询特定策略（cc_strategy、wheel_strategy、default）

### 理解策略要点

从返回结果中学习策略的具体规则，包括但不限于：
- Delta要求和计算方式
- 行权日期选择要求
- 技术指标要求
- 开仓/平仓条件

## 第二步：机会探查和综合分析

### 合理应用工具

基于理解的策略规则，合理调用工具获取所需决策数据：
- 选择期权合约的要求：开仓时间偏好：DTE ≤ 45 天（尽量避免超过两个月）。
- 对比至少对比5个以上不同到期日的期权合约，考虑不同到期日的隐含波动率、Delta值、行权价、年化收益率等因素。（必须同时包含月度期权和周期权）

### 综合分析

综合考虑股票技术指标、VIX恐慌指数、期权链数据、持仓信息、期权成交量等，希腊字母、年化收益率进行综合分析：
- 考虑股票价格、持仓信息（如当前持仓量、持仓delta等）、股票成本、指派情况等因素。
- 考虑期权合约的隐含波动率、Delta值、Gamma值、Theta值、Vega值、Rho值等因素。
- 考虑期权的成交量、流动性、权利金年化收益率等因素，避免成交量为0的标的。

## 第三步：提供交易决策

### 交易决策

根据综合分析结果，提供交易建议：
- 考虑风险收益比，优先选择风险较低 + 年化较高的期权合约。
- 综合考虑权利金收益、时间价值衰减、风险收益比、期权流动性等。
- 提供综合分析报告和推荐标的及选择原因。

## 特殊说明

### 分析 Sell Call/Put 的核心原则

- **至少通过`queryOptionsExpDate`获取至少`6`个到期日，然后使用`queryOptionsChain`查询`PUT`或`CALL`期权链进行分析**
  - 选择的期权链中至少要包含最近`2`个`月度`权链链
  - 必须要对比多个不同到期日的期权合约

## 策略规则优先原则

- 不同策略有不同的delta目标、行权价选择标准，不可一概而论
- 策略规则是决策的依据，工具调用服务于策略执行

## 期权交易关键规则

### 年化收益率计算规则

- **期权合约年化收益率计算公式**：(期权合约价格 × 100) / 期权DTE × 365 / 股价 × 100
  - **示例**：
    - 股价 $100，期权价格 $2，DTE = 30天
    - 年化 = (2 × 100) / 30 × 365 / 100 × 100 = 24.3%

### 卖出期权挂单规则

- **卖出期权使用 Ask 价格挂单**
- 以卖方报价挂单，提高收益率，给用户提供调节可能性

### 止盈平仓价格规则

- 卖出期权止盈平仓时，买回价格应为卖价的 **10%-20%**
- 挂限价单，不是市价单
  - 示例：卖价 $0.39，止盈平仓价 $0.039-$0.078

### 止损平仓时机规则

- 买入期权止损平仓时，不可以提前挂单，只用提前提醒用户。

### 止盈平仓单完整规则

- 止盈价格 = 卖价 × 10%-20%
- 止盈截止时间 (cannelTime) = 期权订单创建时间 + (期权到期日 - 期权订单创建时间) / 2
- 示例：期权3月2日成交，卖价$1，到期日3月26日 → 止盈价$0.15，cannelTime设为3月14日
- 逻辑：一半时间盈利80%-90%，表现良好可取消等待更好机会

### 平仓单检查规则

- 创建平仓单之前，必须先检查是否已有平仓单
- 避免重复创建导致"持仓不足"错误
- 检查方法：调用 queryStrategyDetailAndOrders 查看策略订单列表

### Delta计算规则

- **期权lotSize**：1张期权合约对应的股票数量
  - 重要说明：期权lotSize因标的物、市场和合约规格而异，在实际计算中必须通过工具查询获取准确值。
  - 举例说明：例如美股默认为100。
- **股票delta**：股票delta = 持有股票数 / 期权lotSize
  - 举例说明：例如100股BABA股票的delta = 100（持有股数）/100（期权lotSize） = 1, 100股股票delta为1。
- **期权delta**：每张期权合约都有对应的delta，买入为正数，卖出为负数
  - 举例说明：例如一张ATM delta0.45的期权，买入时delta=0.45，卖出时delta=-0.45。
- **策略delta**：策略delta = 持有股票数 / 期权lotSize + 持有合约数 * 期权delta
  - 举例说明：例如持有100股BABA股票，卖出1张ATM附近delta=0.45的期权合约，那么策略delta就是 `(100/100) + (1*-0.45) = 0.55`。
- **策略delta（归一化）**：策略delta（归一化） = 策略delta / (持股数 / 期权lotSize)
  - 目的说明：归一化策略delta用于标准化不同头寸规模的delta暴露，便于在不同持仓量间进行风险比较和风险管理。
  - 举例说明：例如持有200股BABA股票，卖出2张ATM附近delta=0.45的期权合约，那么策略delta就是 `(200/100) + (2*-0.45) = 1.1`，归一化后策略delta = 1.1 / (200/100) = 0.55。
