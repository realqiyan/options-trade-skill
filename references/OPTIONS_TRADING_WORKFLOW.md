# 期权策略交易流程

当用户涉及期权交易、期权机会相关问题时，使用此流程。

## 第一步：查询策略详细信息和交易订单

所有期权交易、期权机会相关问题，**必须先查询到策略详细信息和交易订单**：

### 查找交易策略

- 如果用户没有提供具体的策略，通过 `queryAllStrategy` 查询所有策略
- 根据用户描述（策略名称、标的代码等）匹配策略
- 无法匹配时需要用户确认具体的策略

### 获取策略详情和订单

- 调用 `queryStrategyDetailAndOrders` 查询策略详细信息和交易订单
- **分析订单组（groupId）**：相同groupId的订单必须合并计算收益（详见"组合订单收益计算"章节）

### 获取策略规则

- 调用 `queryStrategyRule` 查询策略规则
- 根据返回的策略规则内容执行交易决策

## 第二步：机会探查和综合分析

### 获取期权链数据

- 调用 `queryOptionsExpDate` 选取至少 **6个** 到期日
- 调用 `queryOptionsChain` 查询期权链，至少包含最近 **2个月度期权**
- 必须对比多个不同到期日的期权合约

### 综合分析维度

| 分析维度 | 关注指标 |
|---------|---------|
| 股票分析 | 价格、持仓、成本、技术指标 |
| 市场情绪 | VIX恐慌指数 |
| 期权数据 | Delta、Gamma、Theta、Vega、隐含波动率 |
| 流动性 | 成交量、未平仓数（避免成交量为0） |
| 收益 | 权利金年化收益率 |

### 分析近期订单操作

**在给出任何调整建议之前，必须分析近期订单**：

- 检查订单组（orderGroups）：关注最近7天内的订单操作
- 识别Roll操作：发现平仓+开仓组合 = 刚完成展期
- 展期冷却期：期权持仓是最近7天刚展期的，谨慎建议再次展期

## 第三步：提供交易决策

### 交易建议原则

- 考虑风险收益比，优先选择 **风险较低 + 年化较高** 的期权合约
- 综合考虑权利金收益、时间价值衰减、风险收益比、期权流动性
- 提供综合分析报告和推荐标的及选择原因

## 期权交易计算规则

以下规则为期权交易的核心计算方法，必须在交易决策中正确应用。

### Delta计算规则

#### lotSize（期权合约乘数）

- 1张期权合约对应的股票数量
- 美股默认为100
- 实际计算时必须通过工具查询获取准确值

#### 股票Delta

```
股票Delta = 持有股票数 / 期权lotSize
```

**示例**：持有100股BABA，lotSize=100
- 股票Delta = 100 / 100 = 1

#### 期权Delta

- 每张期权合约都有对应的delta值
- 买入期权：delta为正数
- 卖出期权：delta为负数

**示例**：卖出1张delta=0.45的ATM期权
- 期权Delta = 1 × (-0.45) = -0.45

#### 策略Delta

```
策略Delta = 股票Delta + 期权Delta
         = (持有股票数 / lotSize) + (持有合约数 × 期权delta)
```

**示例**：持有100股BABA，卖出1张delta=0.45的期权
- 策略Delta = (100/100) + (1 × -0.45) = 1 - 0.45 = 0.55

#### 策略Delta（归一化）

```
归一化策略Delta = 策略Delta / (持股数 / lotSize)
```

**用途**：标准化不同头寸规模的delta暴露，便于风险比较

### 年化收益率计算

**公式**：
```
年化收益率 = (期权价格 × lotSize) / DTE × 365 / 股价 × 100%
```

**示例**：
- BABA股票：lotSize 100，股价 $100，期权价格 $2，DTE = 30天
- 年化 = (2 × 100) / 30 × 365 / 100 × 100% = 24.3%

### 组合订单收益计算

#### groupId 相同的订单组

当多笔订单具有相同的 `groupId` 时，表示属于同一笔交易操作（如Roll操作）。

**计算规则**：
```
组合净收益 = Σ 该组所有订单的收益之和
组合手续费 = Σ 该组所有订单的手续费之和
```

**重要**：不可单独看某一笔订单判断整体盈亏

#### Roll操作示例

**场景**：Roll Out Sell Put

| 订单类型 | 操作 | 收益 |
|---------|------|------|
| 平仓单 | 买入PUT平仓 | -$100 |
| 新订单 | 卖出新PUT | +$95 |
| **组合** | **净收益** | **-$5** |

- groupId：两笔订单相同
- ❌ 错误：只看新订单认为盈利$95
- ✅ 正确：合并计算，实际亏损$5

### 挂单价格规则

| 操作 | 价格选择 | 原因 |
|-----|---------|------|
| 卖出期权 | Ask 价格 | 以卖方报价挂单，提高收益率 |
| 买入期权 | Bid 价格 | 以买方报价挂单，降低成本 |

### 止盈止损规则

#### 止盈平仓（卖出期权）

- 止盈价格 = 卖价 × **10%-20%**
- 使用限价单，不是市价单

**示例**：卖价 $0.39 → 止盈价 $0.039 ~ $0.078

#### 止盈截止时间

```
止盈截止时间 = 订单创建时间 + (到期日 - 订单创建时间) / 2
```

**示例**：3月2日成交，到期日3月26日 → 止盈截止时间 3月14日

**逻辑**：一半时间盈利80%-90%，表现良好可取消等待更好机会

#### 止损平仓（买入期权）

- 不可以提前挂单
- 只能提前提醒用户

---

## 策略规则优先原则

- 不同策略有不同的delta目标、行权价选择标准，不可一概而论
- 策略规则是决策的依据，工具调用服务于策略执行
- 具体策略规则通过 `queryStrategyRule` 工具动态获取
